7.2 Не работает поиск

27 ноября 2025 г.

15:29

 

Похоже, что указан не корректный адрес ApplicationServer в конфигурации
портала.

Чтобы проверить и изменить на корректное значение, нужно через командную
строку (под администратором) зайти в каталог
"C:\Windows\Microsoft.NET\Framework\v4.0.30319"

и выполнить команду для расшифровки конфигурации портала:

 .\aspnet_regiis -pd "connectionStrings" -app "/ApiServer08042024"

После этого, зайти в каталог  «C:\inetpub\wwwroot\ApiServer08042024»,
открыть файл web.config и в нем, в блоке \<connectionStrings\>,
проверить строку \<add name="AppServerConnection" (параметр
BM.AppServer.Client\|url).

Если указан не корректный адрес Application server - нужно поменять его.

После этого, нужно зашифровать файл web.config командой:

.\aspnet_regiis -pe "connectionStrings" -app "/ApiServer08042024"

и перезапустить IIS.

-----

ou can decrypt the web.config file running:

.\aspnet_regiis -pd "connectionStrings" -app "/ApiServer" 

you can find the utility in c:\windows\Microsoft.Net\Framework\V4.xxxx 
of the server running the IIS with the Apiserver.

Make a backup before trying your settings.

After modification remember to encrypt the values changing -pd by -pe
