- Title  
  How to increase the web portal timeout duration?

- Description  
  To increase the amount of time before a user is automatically logged
  out of their IT Shop session, the web.config file for the web project
  will need to be manually modified.

- Cause  
  By default ASP.NET uses cookies to identify which requests belong to a
  particular session.  
    
  If cookies are not available, a session can be tracked by adding a
  session identifier to the URL.  
    
  To disable cookies, set sessionState cookieless="true".

- Resolution  
  Open the web.config file (e.g. C:\inetpub\wwwroot\IdentityManager)
  using a text editor and increase the timeout in the "sessionState"
  section to the desired amount of time, e.g.:  
    
  sessionState mode="InProc"
  stateConnectionString="tcpip=[REDACTED_IP]:42424"
  sqlConnectionString="data source=[REDACTED_IP];Trusted_Connection=yes"
  cookieless="false" **timeout="30"  
    **
  (where the default is 20)
