Create Dynamic Role

Нажать на значек БД и в поле Condition указать 1=1 - означает, что все пользователи будут иметь доступ к IT Shop

IT Shop используется для административных целей.

Чтобы сделать представление в виде стандартного веб портала, нужна структура каталогов.

В структуре IT Shop создается структура.

Shelf - уровень и на него можно назначить продукт - ресурс (Assigment resource, который прикрепляется к Бизнес роли) (что будет запрашивать пользователь), добавление в роль, например.

Ресурс сначала надо создать (Create Assigment resource в Бизнес ролях) затем Add resources в IT Shop.

Создается Approval policy, в которой делается Workflow по проверкам, которые должны пройти перед тем, как продукт назначится.

На Shelf назначает созданная Approval policy по которой делаются проверки (запросы руководителю и тп.)

Для IT Shop создается сервисная категория (IT Shop -> Basic conf data -> Service category), на нее назначается Service item (IT Shop -> Service catalog -> Custom defined), которой соответствует Бизнеc роль.

Service item создается, через Business Roles -> выбрать роль -> Create assignment resource.

То есть, мы на бизнес роль назначаем ресурс, который потом закрепляем за Shelf. Затем закрепляем Service Item за Service Catalog

-Чтобы назначить группу AD на IT Shelf

Создать группу AD, в OneIM и поставить галку IT Shop!

Обязательно выбрать Service Item для нее!

После этого, назначить эту AD группу на Shelf.

Нужно разместить Service item в Service catalog, чтобы он отображался на портале.

Если на портале у сотрудника нет галки для запроса продукта - надо создать Approval policy и назначть на shelf ее.

Проблема была в том, что я при назначении группы в self создавал ресурс (Service item) с тем же именем, что и когда создавал ресурс (Service item) для бизнес роли.

Так как Shelf связвается с группой или бизнес ролью, через Service item!

Пользователь alexb делает запрос.

У него Manager - IARLOFS (он сначала должен аппрувить запрос его (CM))

Alexb в дапартаменте Группа учета налога - у нее менеджер IARLOSS. Он тоже должен аппрувить запрос. После этого изначальный запрос будет аппрувлен и у alexb появятся запрошенные права.

Разграничение прав просмотра Service category для пользователей портала.

Можно реализовать, через несколько ITShops, с динамическим назначением пользователей на них.

Напоминания настраиваются в шаге Workflow

Выполнение скрипта в процессе работы.

Смысл в том, что вызывается событие в процессе, а на это событие срабатывает запуск скрипта.
